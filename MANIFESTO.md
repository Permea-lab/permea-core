# Permea Manifesto

Permea Core is being built as a public technical foundation for sequence-first delivery and mRNA expression engineering. The repository exists to make benchmark tasks, technical assumptions, and reproducible workflows inspectable from the start.

## Principles

- Open-source-first where legal, ethical, biosafety, and confidentiality constraints permit.
- Benchmark-first before broader platform expansion.
- Reproducible workflows over presentation-driven outputs.
- Clear interfaces between research framing, implementation, evaluation, and results.
- Public documentation of scope, assumptions, and constraints.

## Operating Commitments

- The repository should define what is being measured before it claims anything about performance.
- Benchmark logic should remain separable from model-specific implementation.
- Results should only be treated as canonical when they are tied to explicit provenance.
- Documentation should function as part of the system contract, not as an afterthought.

## Boundaries

Permea Core is not a claim of validated biology. It is an open toolkit and benchmarks program for building a disciplined technical base around sequence-first delivery and mRNA expression engineering.

The project should remain useful to outside technical readers without private context. That requirement is part of the standard, not an optional feature.
