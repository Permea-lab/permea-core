# Dataset Lifecycle

Dataset lifecycle status is about public review maturity. It does not imply biological efficacy, experimental validation, therapeutic outcome, clinical evidence, expression improvement, or solved delivery.

## Statuses

| Status | Entry criteria | Required provenance | Required evidence | Allowed public claims | Prohibited claims | Exit criteria |
| --- | --- | --- | --- | --- | --- | --- |
| Proposed | Dataset idea, source category, and intended use are described. | Source type and expected source reference are named if known. | Registry entry or proposal note. | May state that a dataset surface is proposed. | No dataset existence, acquisition completion, active reference, experimental validation, clinical evidence, or solved-delivery claim. | Draft card is created. |
| Draft Card | Dataset card fields are partially populated. | Source reference, intended use, data type, and known limitations are recorded. | Draft card and local review notes. | May state that dataset metadata is being drafted. | No reproducible derived dataset, active reference, independent reproduction, or external validation claim. | Provenance record is complete enough for documented external review. |
| Documented External Dataset | External source is documented without claiming acquisition or redistribution completion. | Source reference, license or usage constraints, acquisition method, and known limitations are recorded. | Dataset card, source registry link where applicable, and validation output. | May state that an external dataset source is documented. | No raw data possession, redistribution-rights confirmation, reproducible derived dataset, or biological outcome claim unless separately supported. | Reproducible processing path or derived artifact record is added. |
| Reproducible Derived Dataset | Public commands or records reproduce derived metadata or generated artifacts. | Acquisition method, processing steps, transformation summary, generated artifacts, checksums where applicable, and reproducibility path are recorded. | Run manifest, generated artifact, validation report, and linked evidence record. | May state that derived dataset artifacts reproduce locally. | No independent reproduction, external validation, experimental validation, or clinical evidence claim. | Maintainer review confirms reference readiness. |
| Active Reference Dataset | Governance review accepts the dataset as a public reference workflow. | Complete provenance record, version, license constraints, checksums where applicable, and maintenance notes are present. | Completed dataset card, reproducibility evidence, validator output, benchmark link, and claim registry link. | May state that the dataset is an active computational reference dataset. | No biological efficacy, therapeutic outcome, expression improvement, clinical evidence, or solved-delivery claim. | Independent reproduction record is added or dataset is revised. |
| Independently Reproduced | Independent reviewer reproduces the documented computational dataset workflow. | Reviewer identity or review context, reproduction date, artifact hashes where applicable, and deviations are recorded. | Dated reproduction report and evidence link. | May state that independent computational reproduction was recorded. | No external validation or experimental validation claim unless separately supported. | External validation record is added or dataset is revised. |
| Externally Validated | External reviewer or source owner validates provenance, use constraints, or reproduction boundary. | External validation context, date, scope, and limitations are recorded. | External validation record and evidence link. | May state the specific external validation scope. | No experimental, clinical, therapeutic, or broad biological outcome claim. | Dataset is maintained, revised, deprecated, or archived. |
| Deprecated | Dataset should no longer be used for new reference work. | Deprecation reason, replacement path if any, and affected links are recorded. | Deprecation notice and registry update. | May state that a dataset is deprecated. | No active reference claim. | Archived or replaced. |
| Archived | Dataset is retained for historical traceability. | Archive location, final version, and limitations are recorded. | Archive record and registry update. | May state that a dataset is archived. | No current-use or active-reference claim. | Reopening requires a new lifecycle review. |

## Review Rule

Promoting a dataset status requires updating the registry, card metadata, provenance record, linked evidence, linked benchmark references, and claim boundaries in the same review.
