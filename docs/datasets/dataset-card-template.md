# Dataset Card Template

Use this template for future Permea dataset cards. Keep claims bounded to the lifecycle status and linked evidence.

## Dataset ID

Stable identifier, for example `example_dataset_id`.

## Name

Human-readable dataset name.

## Purpose

Why this dataset card exists and what review surface it supports.

## Source Type

External dataset, derived dataset, synthetic fixture, internal review surface, proposed source, or archived source.

## Source Reference

Citation, source registry link, accession, URL, release identifier, or other public source reference where applicable.

## Intended Use

Expected use in examples, validation, benchmark support, evidence review, or future reproducibility work.

## Data Type

Sequence, metadata, labels, aggregate metrics, generated artifact, or other bounded data category.

## Inclusion Criteria

Rules for including records or examples.

## Exclusion Criteria

Rules for excluding records or examples.

## Processing Summary

Short description of cleaning, filtering, normalization, transformation, or aggregation steps.

## Provenance Record

Source reference, acquisition method, processing steps, transformation summary, generated artifacts, checksums where applicable, license or usage constraints, reproducibility path, and known limitations.

## License / Usage Constraints

License, redistribution, review, citation, access, and usage constraints.

## Linked Benchmarks

Benchmark IDs or benchmark registry links that can use this dataset card.

## Linked Evidence

Evidence IDs, generated artifacts, validation commands, reports, or reproduction records.

## Linked Claims

Supported claim categories and explicit unsupported claim boundaries.

## Reproducibility Requirements

Commands, inputs, artifact paths, expected outputs, hashes where applicable, and validation status.

## Claim Boundaries

Minimum explicit boundaries:

- no dataset acquisition completion claim unless separately validated
- no redistribution-rights confirmation unless separately documented
- no wet-lab validation by Permea
- no biological efficacy claim
- no therapeutic outcome claim
- no BBB success claim
- no solved-delivery claim
- no SOTA performance claim
- no experimental validation claim
- no clinical evidence claim
- no expression improvement claim

## Limitations

Known source, acquisition, processing, label, benchmark, evidence, usage, and review limitations.

## Status

Lifecycle status from [Dataset lifecycle](dataset-lifecycle.md).

## Version

Dataset card version and, where applicable, source version or release.

## Maintainer Notes

Review notes, pending work, governance decision, or next recommended evidence.
