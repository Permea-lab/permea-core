# Research Package Template

Use this template for future Permea research packages. Keep claims bounded to lifecycle status, linked evidence, and review state.

## Research Package ID

Stable identifier, for example `example_research_package_id`.

## Title

Human-readable package title.

## Purpose

Why the package exists and what review surface it supports.

## Package Type

Paper support package, technical report package, external review packet, reproducible artifact package, proposed package, superseded package, or archive package.

## Intended Audience

Researchers, reviewers, collaborators, investors, partners, maintainers, or public technical readers.

## Summary

Short package overview with bounded scope and no unsupported claims.

## Linked Evidence

Evidence IDs, evidence maps, generated artifacts, reports, review records, or reproduction records.

## Linked Benchmarks

Benchmark IDs, benchmark registry entries, or benchmark cards.

## Linked Datasets

Dataset IDs, dataset registry entries, dataset cards, and provenance records.

## Linked Specifications

Specification records, schemas, artifact specs, and validation contracts.

## Linked Claim Registry Entries

Claim registry entries and explicit unsupported-claim boundaries.

## Reproducibility Path

Commands, inputs, generated outputs, expected status, and reproducibility report links.

## Validation Path

Commands, validators, expected status, and validation report links.

## Figures / Tables Inventory

List figure and table identifiers, source artifacts, generation paths, and limitations. Mark missing or proposed figures as not yet demonstrated.

## Limitations

Known evidence, benchmark, dataset, specification, validation, reproducibility, review, audience, and claim limitations.

## Claim Boundaries

Minimum explicit boundaries:

- no paper claim unless separately reviewed and approved
- no new scientific result claim from this layer
- no wet-lab validation by Permea
- no biological efficacy claim
- no therapeutic outcome claim
- no BBB success claim
- no solved-delivery claim
- no SOTA performance claim
- no experimental validation claim
- no clinical evidence claim
- no expression improvement claim

## Status

Lifecycle status from [Research package lifecycle](research-package-lifecycle.md).

## Version

Package version and, where applicable, release or revision identifier.

## Maintainer Notes

Review notes, pending work, governance decision, or next recommended evidence.
