# Research Package Layer

The research package layer is the public operating surface for assembling Permea Core evidence, benchmarks, datasets, specifications, and claim boundaries into reviewable packages.

It explains how Permea turns evidence assets, benchmark cards, dataset cards, specification records, validation outputs, reproducibility outputs, and claim registry entries into structured packages for papers, technical reports, external review packets, or reproducible public artifacts.

## Review Path

Start with the deterministic CLI:

```bash
python3 scripts/permea_research.py
```

## Status

Research Package Framework Ready

This layer does not write a new paper, submit a paper, publish a paper, create a technical report with new results, or claim new scientific findings. Current entries are proposed package surfaces for future evidence packaging.

## Claim Boundaries

Research packages may organize computational evidence, reproducibility paths, validation commands, and claim boundaries. They do not establish biological effect, experimental validation, therapeutic outcome, clinical evidence, expression improvement, or solved delivery.

Explicit non-claims:

- no paper claim unless separately reviewed and approved
- no new scientific result claim from this layer
- no wet-lab validation by Permea
- no biological efficacy claim
- no therapeutic outcome claim
- no BBB success claim
- no solved-delivery claim
- no SOTA performance claim
- no experimental validation claim
- no clinical evidence claim
- no expression improvement claim
