# DEC-006: Project Memory Layer Required for Continuation

## Context

Permea Core has multiple public documentation and evidence surfaces. Future reviewers and contributors need a current path into the repository without relying on historical chat logs or unstated context.

## Decision

Permea Core will maintain a project memory layer through `OPEN_THIS_FIRST.md`, `REVIEW_HUB.md`, ADRs, runbooks, reports, and decision records.

## Rationale

The memory layer keeps the project self-documenting. It separates current state from historical task detail and gives future work a reliable continuation path.

## Alternatives Considered

- Rely only on reports.
- Rely only on generated evidence surfaces.
- Rely on contributor memory or issue history.

These alternatives were rejected because they make continuation fragile and increase the chance of repeated strategic debate.

## Consequences

- Major tasks should update the right memory surface for the work performed.
- Strategic or technical decisions should create or update decision records.
- The review hub should remain the navigation map, not a full project history.
