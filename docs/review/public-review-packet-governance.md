# Public Review Packet Governance

Public review packet governance defines how Permea Core public review packets are proposed, updated, checked, superseded, and archived.

## Proposal

A packet may be proposed when it has a clear intended reviewer, review objective, required reading path, first command, linked evidence, linked registries, linked claims, validation path, unsupported claims, and limitations.

## Updates

Update a public review packet when evidence, benchmark, dataset, research package, claim, validation, reproducibility, or review surfaces change.

## Evidence Link Checks

Evidence links should be checked for:

- current file path
- evidence status
- related reports
- related validators
- related claim boundaries
- limitations

## Claim Boundary Enforcement

Every packet must preserve explicit unsupported-claim language and must not imply new scientific evidence, paper claims, wet-lab validation by Permea, biological efficacy, therapeutic outcomes, BBB success, SOTA performance, experimental validation, clinical evidence, expression improvement, or solved delivery.

## Unsupported Claims

Unsupported claims must be listed directly or linked to a claim registry entry. Future-facing items must be marked proposed, not yet demonstrated, out of scope, or requiring future validation.

## Superseded Or Archived Packets

Supersede a packet when a newer review path replaces it. Archive a packet when it is retained only for historical traceability.

Superseded and archived packets should retain:

- prior packet ID
- successor packet if any
- reason for replacement
- final validation status
- final claim-boundary status
- known limitations
