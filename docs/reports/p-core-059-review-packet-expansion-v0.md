# P-CORE-059 Review Packet Expansion v0

## Summary

P-CORE-059 expands the Evidence Review Packet System from the initial P-CORE-053 artifact consistency packet to three additional public artifact surfaces:

- P-CORE-030 evidence surface layer
- P-CORE-032 reproducibility bundle
- P-CORE-034 evaluation bundle

The expansion keeps the packet model generator-backed. Packet Markdown and JSON files are generated by `python3 scripts/permea_review_packet.py` from definitions in `src/permea_core/review_packets/packets.py`.

## Target Selection Rationale

The selected targets are existing public Permea Core artifact systems with concrete source files, generated outputs, evidence records, validation commands, tests, limitations, and claim-boundary language.

| Packet | Primary artifact | Why selected |
| --- | --- | --- |
| P-CORE-030 evidence surface layer | `docs/examples/generated/README.md` | Central generated navigation surface for public artifacts, evidence docs, validation commands, and limitations. |
| P-CORE-032 reproducibility bundle | `docs/examples/generated/REPRODUCIBILITY_REPORT.md` | Reviewable local reproduction report tied to generator code, validation commands, lineage, and focused tests. |
| P-CORE-034 evaluation bundle | `docs/examples/generated/EVALUATION_PACKET.md` | Reviewable template/reference evaluation packet tied to input families, generated outputs, validation handoff, and reproducibility handoff. |

P-CORE-053 remains in the packet set as the original artifact consistency review packet.

## Implementation Notes

- Packet definitions now cover four packet IDs in `src/permea_core/review_packets/packets.py`.
- `docs/review/packets/README.md` and `docs/review/review-packet-system.md` list the expanded packet set.
- `scripts/verify_review_packet_raw_urls.py` derives raw URL targets from `PACKET_OUTPUTS` so every generated packet Markdown and JSON file is checked.
- Regression tests validate deterministic generation, required fields, existing links, physical multiline Markdown, pretty JSON, explicit boundary language, and absence of prohibited private/internal references.

## Claim Boundaries

This report and the generated packets are reviewability surfaces only.

They do not create scientific evidence, new benchmark results, biological validation, wet-lab validation, clinical efficacy evidence, model performance evidence, or solved-delivery evidence.

They do not add non-public operational details or private-only operating material.

## Limitations

- The packet set is curated for selected high-value public artifact systems, not every repository file.
- Passing validation means packet files are deterministic and reviewable; it does not mean the linked artifact systems prove biological outcomes.
- Raw URL verification depends on pushed branch contents and should be rerun after each packet-changing push.
