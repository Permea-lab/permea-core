# Support

## Where to Ask Questions

Use repository issues or discussions for public, non-sensitive questions about Permea Core.

Appropriate public questions include:

- documentation clarification
- benchmark contract interpretation
- result artifact schema questions
- run manifest and provenance questions
- reproducibility workflow questions
- contributor workflow questions
- claim hygiene questions

Do not post secrets, credentials, private data, restricted row-level datasets, unlicensed biological datasets, or confidential partner information.

For sensitive disclosures, see [SECURITY.md](SECURITY.md).

## What Is Supported

Permea Core can support questions and contributions about:

- project documentation
- benchmark contracts
- result artifact schemas
- run manifest schemas
- reproducibility expectations
- dataset and provenance policy
- public-safe artifact boundaries
- claim hygiene and source-to-claim review
- contributor and community workflow

## What Is Not Supported

Permea Core does not provide:

- medical advice
- clinical advice
- therapeutic recommendations
- guarantee of wet-lab validation
- guarantee of therapeutic efficacy
- guarantee of clinical efficacy
- confirmation that dataset redistribution is permitted without source/license review
- private consulting through public issue threads

Permea may use "AlphaFold for Delivery" as an ambition or positioning phrase only. It must not be read as a claim of AlphaFold-level performance, adoption, or standardization.

## Issue Categories

When opening an issue, choose the closest category:

- bug
- docs
- benchmark contract
- result artifact schema
- dataset/provenance
- reproducibility
- claim hygiene
- contributor/community
- security or sensitive disclosure

For security or sensitive disclosure issues, avoid public details and follow [SECURITY.md](SECURITY.md).

## Project Status

Permea Core is an early open scientific OSS project.

The current project focus is:

- clear benchmark definitions
- reproducible workflows
- provenance-aware result artifacts
- evidence-bounded interpretation
- contributor and governance foundations

Public artifacts are limited by dataset, source, license, and release policy. Row-level and source-derived artifacts may require review before public release.

## Claim Boundaries

Permea must not claim:

- completed wet-lab validation unless documented
- clinical efficacy
- therapeutic efficacy
- universal delivery prediction
- production-grade drug delivery platform status
- dataset redistribution permission without source/license approval

Benchmark claims must remain scoped to dataset, split, metric, and evidence level.
